# BS2PRO OTA 固件蓝牙命令分析

本文档基于 `ota/CH591_For_BS2PRO_Ver0.0.3.5.bin` 及其反汇编产物分析，目标是整理固件中可识别的蓝牙协议帧、命令分发表、常用控制命令和设备上报格式。

分析结论来自三类证据：

- 固件反汇编：`ota/dump.S` 中 `0x5e7a` 是 `5A A5` 协议帧入口，`0x58fe` 是命令分发表，`0x881e` / `0x8840` 是校验计算/填充逻辑。
- 现有抓包：`scripts/ble_data.md` 记录了 BLE ATT Write Command，经由 handle `0x003c` 写入 `5A A5 ...` 帧。
- 现有控制代码：`internal/device/device.go`、`internal/device/rgb.go`、`internal/types/types.go` 已实测使用多条同协议命令。

> 注意：固件命令分发表只能证明固件识别这些 `CMD` 分支；部分内部/调试命令缺少完整业务上下文，本文按置信度标注。未确认命令不建议直接对真实设备批量 fuzz。

## 传输层与 GATT

BS2PRO 的蓝牙写入层使用 BLE GATT Write Command / Write Without Response。已有抓包显示：

| 方向 | ATT | Handle | Value |
| --- | --- | --- | --- |
| 主机到设备 | Write Command `0x52` | `0x003c` | `5A A5 ...` |
| 设备到主机 | Notify | 抓包/脚本中常见 `0x0026` 或 `0000fff1-0000-1000-8000-00805f9b34fb` | `5A A5 EF ...` |

仓库脚本中曾发现通知特征 UUID 为 `0000fff1-0000-1000-8000-00805f9b34fb`。写入特征在同类设备上通常与 `FFF0/FFF1/FFF2` 私有服务族相关，但 BS2PRO 的可写特征以实际枚举结果和抓包 handle 为准。

HID 模式与 BLE payload 共用 `5A A5` 内层协议，但 HID 写入前面会额外加 Report ID `0x02` 并补齐到报告长度；BLE GATT 写入不带 Report ID。

## 帧格式

### 主机发送帧

```text
5A A5 <CMD> <LEN> <PAYLOAD...> <CHECKSUM>
```

字段含义：

| 字段 | 长度 | 说明 |
| --- | --- | --- |
| `5A A5` | 2 | 固定同步头 |
| `CMD` | 1 | 命令码 |
| `LEN` | 1 | 从 `CMD` 开始到 payload 结束的长度，即 `2 + payload_len` |
| `PAYLOAD` | `LEN - 2` | 命令参数 |
| `CHECKSUM` | 1 | `sum(CMD, LEN, PAYLOAD...) & 0xFF` |

示例：设置目标转速 1700 RPM。

```text
5A A5 21 04 A4 06 CF
```

计算：

```text
CMD=21
LEN=04
PAYLOAD=A4 06    # 0x06A4 = 1700, little endian
CHECKSUM=(21 + 04 + A4 + 06) & FF = CF
```

旧脚本中有一处把校验写成“包含 `5A A5` 再 `+1`”。两者在该协议下等价，因为 `0x5A + 0xA5 + 1 = 0x100`，低 8 位为 `0x00`。

### 设备通知帧

设备主动状态上报使用：

```text
5A A5 EF 0B <gear_settings> <work_mode> <reserved>
<current_rpm_le> <target_rpm_le> <extra...>
```

BLE 通知没有 HID Report ID。HID 读取时前面可能多一个 Report ID `0x01`。

字段解析：

| Offset(BLE) | 长度 | 字段 | 说明 |
| --- | --- | --- | --- |
| `0..1` | 2 | `5A A5` | 固定头 |
| `2` | 1 | `EF` | 状态上报命令码 |
| `3` | 1 | `0B` | 上报长度/状态类型，常见固定值 |
| `4` | 1 | `gear_settings` | 高 4 位为最高挡位，低 4 位为设置挡位 |
| `5` | 1 | `work_mode` | 常见 `0x04` 挡位模式，`0x05` 实时/自动转速模式 |
| `6` | 1 | reserved | 常见 `0x05` |
| `7..8` | 2 | current RPM | 小端 |
| `9..10` | 2 | target RPM | 小端 |
| `11..` | n | extra | 设备内部计数/状态，未完全确认 |

`gear_settings` 编码：

| 半字节 | 值 | 含义 |
| --- | --- | --- |
| 高 4 位 | `0x2` | 最高挡位标准 |
| 高 4 位 | `0x4` | 最高挡位强劲 |
| 高 4 位 | `0x6` | 最高挡位超频 |
| 低 4 位 | `0x8` | 设置挡位静音 |
| 低 4 位 | `0xA` | 设置挡位标准 |
| 低 4 位 | `0xC` | 设置挡位强劲 |
| 低 4 位 | `0xE` | 设置挡位超频 |

## 固件命令总表

以下命令码来自固件 `0x58fe` 分发表。置信度说明：

- 高：固件分支明确，且仓库代码或抓包已使用。
- 中：固件分支明确，业务语义可由分支和邻近代码推断，但缺少完整抓包验证。
- 低：固件分支明确，但看起来偏内部、调试或状态机控制，不建议直接调用。

| CMD | LEN/参数 | 功能 | 置信度 | 备注 |
| --- | --- | --- | --- | --- |
| `0x01` | 无 | 查询/返回固定设备信息块 | 中 | 构造一段固定长度回包，偏信息查询 |
| `0x02` | 无 | 查询当前配置摘要 | 中 | 回包含当前状态字节 |
| `0x03` | 无 | 初始化/启动运行态资源 | 低 | 会置内部运行标志并触发定时/状态更新 |
| `0x04` | 无 | 查询/上报系统配置快照 | 低 | 回包形态由固件固定构造，业务未完全确认 |
| `0x05` | 无 | 清除/复位某运行状态 | 低 | 写内部状态位 |
| `0x06` | 无 | 重新初始化多组运行参数 | 低 | 调用多段初始化函数 |
| `0x07` | 无 | 查询某状态字段 | 低 | 返回内部 offset `0x0C` 字段 |
| `0x08` | `gear` | 固定挡位设置 | 高 | `01` 静音，`02` 标准，`03` 强劲，`04` 超频 |
| `0x09` | 无 | 查询一个动态状态字节 | 低 | 与内部参数块相关 |
| `0x0A` | `value` | 设置一个内部开关/状态 | 低 | 返回成功/失败 |
| `0x0B` | 无 | 返回一段 18 字节左右的固定/设备信息 | 低 | 类似查询 |
| `0x0C` | `mode` | 通电自启动 | 高 | `01` 开，`02` 关 |
| `0x0D` | `mode` | 智能启停 | 高 | `00` 关，`01` 立即，`02` 延时 |
| `0x21` | `rpm_lo rpm_hi` | 实时目标转速 | 高 | 小端 RPM；建议先发 `0x23` |
| `0x22` | 不明 | 未完整实现/异常分支 | 低 | 分支很短，不建议使用 |
| `0x23` | 无 | 进入实时转速控制模式 | 高 | 常作为设置 `0x21` 前置命令 |
| `0x24` | 无 | 退出/停止实时转速控制模式 | 中 | 会清除实时模式标志 |
| `0x25` | 无 | 查询当前工作状态/挡位状态 | 中 | 返回当前模式或挡位字段 |
| `0x26` | `gear_idx rpm_lo rpm_hi` | 设置挡位档案转速 | 高 | `gear_idx` 为 `00..03`，固件会写入对应挡位转速表 |
| `0x27` | 无 | 查询挡位转速表/预设表 | 中 | 返回多组转速配置 |
| `0x41` | 无 | 灯效数据传输初始化 | 高 | RGB 上传流程握手之一 |
| `0x42` | data | 写入灯效数据缓冲 | 高 | 用于分块传输灯效帧数据 |
| `0x43` | 无或 `01` | 灯效提交/应用/结束阶段 | 高 | 常用于应用已上传灯效或关灯流程收尾 |
| `0x44` | `mode` | 灯效模式/动态参数 | 高 | `01` 智能温控灯效相关；`02` 可能为 RGB 参数 |
| `0x45` | 无 | 灯效/动态模式心跳或状态查询 | 高 | 仓库中作为维持/握手包使用 |
| `0x46` | `enable` | 灯效开关/动态灯效入口 | 高 | `00` 关，`01` 开 |
| `0x47` | `frame_idx + 10 bytes` | 写入灯效帧 | 高 | `frame_idx=00` 为头部参数，`01..1E` 为 30 帧数据 |
| `0x48` | `enable` | 挡位灯开关 | 高 | `00` 关，`01` 开 |
| `0xED` | 无 | 关闭内部调试/上报标志 A | 低 | 固件内部标志，非业务命令 |
| `0xEE` | 无 | 打开内部调试/上报标志 A | 低 | 固件内部标志，非业务命令 |
| `0xF0` | 无 | 触发内部配置/状态刷新 | 低 | 调用与 `0x04` 相同的配置构造路径 |
| `0xF1` | 无 | 关闭校验/调试标志 B | 低 | 影响协议校验/调试分支，不建议使用 |
| `0xF2` | 无 | 打开校验/调试标志 B | 低 | 影响协议校验/调试分支，不建议使用 |
| `0xEF` | 通知上报 | 设备状态上报 | 高 | 设备到主机方向，不是常规主机写命令 |

## 常用控制命令

### 实时转速控制

进入实时转速模式：

```text
5A A5 23 02 25
```

设置目标 RPM：

```text
5A A5 21 04 <rpm_lo> <rpm_hi> <checksum>
```

示例：

| RPM | 小端参数 | 完整 BLE Value |
| --- | --- | --- |
| 1700 | `A4 06` | `5A A5 21 04 A4 06 CF` |
| 3300 | `E4 0C` | `5A A5 21 04 E4 0C 15` |
| 3981 | `8D 0F` | `5A A5 21 04 8D 0F C1` |

退出实时转速模式：

```text
5A A5 24 02 26
```

### 固定挡位

BS2PRO 固件同时支持简单挡位命令 `0x08` 和更细的挡位档案转速命令 `0x26`。

简单固定挡位：

| 挡位 | 命令 |
| --- | --- |
| 静音 | `5A A5 08 03 01 0C` |
| 标准 | `5A A5 08 03 02 0D` |
| 强劲 | `5A A5 08 03 03 0E` |
| 超频 | `5A A5 08 03 04 0F` |

细分挡位档案设置：

```text
5A A5 26 05 <gear_idx> <rpm_lo> <rpm_hi> <checksum>
```

仓库中已记录的档案命令：

| 档案 | RPM | BLE Value |
| --- | --- | --- |
| 1挡低 | 1300 | `5A A5 26 05 00 14 05 44` |
| 1挡中 | 1700 | `5A A5 26 05 00 A4 06 D5` |
| 1挡高 | 1900 | `5A A5 26 05 00 6C 07 9E` |
| 2挡低 | 2100 | `5A A5 26 05 01 34 08 68` |
| 2挡中 | 2400 | `5A A5 26 05 01 60 09 95` |
| 2挡高 | 2700 | `5A A5 26 05 01 8C 0A C2` |
| 3挡低 | 2800 | `5A A5 26 05 02 F0 0A 27` |
| 3挡中 | 3000 | `5A A5 26 05 02 B8 0B F0` |
| 3挡高 | 3300 | `5A A5 26 05 02 E4 0C 1D` |
| 4挡低 | 3500 | `5A A5 26 05 03 AC 0D E7` |
| 4挡中 | 3700 | `5A A5 26 05 03 74 0E B0` |
| 4挡高 | 4000 | `5A A5 26 05 03 A0 0F DD` |

> 说明：仓库旧注释中有个别 RPM 文字和十六进制值不一致，表中 RPM 按小端参数计算。例如 `60 09` 实际为 `0x0960 = 2400`。

## RGB / 灯效命令

灯效命令的校验范围同样是 `CMD/LEN/PAYLOAD`。仓库实现中的通用构造为：

```text
5A A5 <fields...> <sum(fields) & FF>
```

其中 `fields[0]` 就是 `CMD`，`fields[1]` 是 `LEN`。

### 基础灯效命令

| 功能 | BLE Value | 说明 |
| --- | --- | --- |
| 关闭 RGB | `5A A5 46 03 00 49` | `0x46 enable=0` |
| 开启/进入动态灯效 | `5A A5 46 03 01 4A` | `0x46 enable=1` |
| 灯效心跳/查询 | `5A A5 45 02 47` | 上传流程握手 |
| 灯效参数 1 | `5A A5 44 03 01 48` | 智能温控灯效常用 |
| 灯效参数 2 | `5A A5 44 03 02 49` | RGB 参数，语义未完全命名 |
| 应用/提交 | `5A A5 43 03 01 47` | 应用灯效帧 |
| 亮度 100% 快速应用 | `5A A5 43 02 45` | 旧代码用于亮度 100% |

### 完整灯效上传流程

仓库 `internal/device/rgb.go` 中的完整上传序列：

```text
5A A5 46 03 00 49    # 先关闭/清空
5A A5 46 03 01 4A
5A A5 46 03 01 4A
5A A5 45 02 47
5A A5 45 03 01 49
5A A5 41 02 43
5A A5 41 03 01 45
5A A5 47 0D 00 <10-byte f0> <checksum>
5A A5 47 0D 01 <10-byte frame01> <checksum>
...
5A A5 47 0D 1E <10-byte frame30> <checksum>
5A A5 43 03 01 47
```

`0x47` 的 `LEN=0D`，payload 为 `frame_index + 10 bytes`。

`f0` 参数结构按仓库实现为：

```text
00 02 00 <mode> <speed> <brightness> <R> <G> <B> 00
```

速度常用编码：

| 速度 | 值 |
| --- | --- |
| fast | `0x05` |
| medium | `0x0A` |
| slow | `0x0F` |

已实现灯效模式：

| 模式 | `mode` / 数据特点 |
| --- | --- |
| 关闭 | `0x46 03 00` |
| 智能温控 | `46/45/44/43` 握手，不上传 30 帧 |
| 单色静态 | `f0 mode=00`，帧中写固定灯珠颜色 |
| 多色静态 | `f0 mode=00`，帧中写多组颜色 |
| 旋转 | `f0 mode=05`，上传 304 字节色流 |
| 流动 | `f0 mode=05`，按内置流动模板上传 |
| 呼吸 | `f0 mode=len(colors)*2-1`，按颜色数量构造周期数据 |

### 亮度命令

旧 HID/BLE 记录中存在两个快捷亮度命令：

| 功能 | BLE Value | 说明 |
| --- | --- | --- |
| 0% | `5A A5 47 0D 1C 00 FF 00 00 00 00 00 00 00 00 6F` | 本质是向 `0x47` 写一帧特殊亮度/关闭数据 |
| 100% | `5A A5 43 02 45` | 快捷应用/提交命令 |

更完整的亮度控制建议使用完整灯效帧上传，把 `f0[5]` 设为 `0..100` 并按比例写 RGB 帧。

## 查询、状态与内部命令

这些命令在固件中存在明确分支，但业务层未全部实测。建议把它们视为“可研究命令”，不要作为面向用户的稳定 API。

| CMD | 可能用途 | 说明 |
| --- | --- | --- |
| `0x01` | 设备/固件信息查询 | 固件构造固定回包并补校验 |
| `0x02` | 当前配置查询 | 返回一个 6 字节左右的配置摘要 |
| `0x04` / `0xF0` | 状态快照/配置刷新 | 构造固定状态帧 |
| `0x07` / `0x09` / `0x0B` | 状态查询 | 返回内部状态字段或固定信息块 |
| `0x24` | 退出实时模式 | 会清除实时转速模式标志 |
| `0x25` | 查询当前工作状态 | 返回当前挡位/模式相关字段 |
| `0x27` | 查询转速档案表 | 返回若干小端转速配置 |
| `0xED` / `0xEE` | 调试标志 A | 开关某内部状态标志 |
| `0xF1` / `0xF2` | 调试/校验标志 B | 会影响校验错误处理路径，不建议使用 |

## 状态上报示例

BLE 通知示例：

```text
5A A5 EF 0B 68 05 05 C4 09 8D 0F A1 01 77 ...
```

解析：

| 字段 | 值 | 说明 |
| --- | --- | --- |
| `EF` | `0xEF` | 状态上报 |
| `0B` | `0x0B` | 上报类型/长度字段 |
| `gear_settings` | `0x68` | 最高挡位 `0x6` 超频，设置挡位 `0x8` 静音 |
| `work_mode` | `0x05` | 实时/自动转速模式 |
| `reserved` | `0x05` | 未知 |
| current RPM | `C4 09` = 2500 | 小端 |
| target RPM | `8D 0F` = 3981 | 小端 |

HID 中同一帧前面会多 `01`：

```text
01 5A A5 EF 0B ...
```

## 生成命令的参考代码

```go
func buildFrame(cmd byte, payload ...byte) []byte {
    ln := byte(len(payload) + 2)
    frame := []byte{0x5A, 0xA5, cmd, ln}
    frame = append(frame, payload...)
    var sum byte
    for _, b := range frame[2:] {
        sum += b
    }
    frame = append(frame, sum)
    return frame
}

func buildRPM(rpm uint16) []byte {
    return buildFrame(0x21, byte(rpm), byte(rpm>>8))
}
```

## 建议的安全调用顺序

实时转速：

```text
5A A5 23 02 25
sleep 50ms
5A A5 21 04 <rpm_lo> <rpm_hi> <checksum>
```

RGB 灯效：

```text
46 03 00 -> 46 03 01 -> 45 02 -> 45 03 01 -> 41 02 -> 41 03 01
-> 47 0D 00/f0 -> 47 0D 01..1E/frame -> 43 03 01
```

配置项：

```text
0C 03 xx    # 通电自启动
0D 03 xx    # 智能启停
48 03 xx    # 挡位灯
```

## 未完全确认点

- GATT 可写特征 UUID 需要以实际设备枚举为准；抓包确认 handle `0x003c` 可写，通知侧常见 `FFF1`。
- `0x01`、`0x02`、`0x04`、`0x07`、`0x09`、`0x0B`、`0x25`、`0x27` 的回包字段还需要更多通知抓包来命名。
- `0xED`、`0xEE`、`0xF1`、`0xF2` 是固件内部/调试标志，不建议在正式控制流程中使用。
